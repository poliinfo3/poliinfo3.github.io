Question Answering

  1. HOME
  2. Tasks
  3. Question Answering

目的

QA Lab-PoliInfo-3 Question Answering は、一括質問一括答弁方式の会議において、会議録と、その質問の要約が与えられたときに、対応する答弁の要約を生成するタスクです。

20210519_PoliInfo3_QA

以前のタスク（PoliInfo Summarization、PoliInfo-2 Dialog Summarization）との主な違いを以下に示します。

  • 答弁の要約のみを対象とします（質問の要約は与えられます）。
  • 同一の答弁者の発言全体が要約元となります。以前のタスクにあった要約元の範囲（StartingLine、EndingLine）は与えられません。なお、要約元の抽出については QA Alignment タスクも参照してください。
  • 複数の質問・答弁をまとめて要約していた箇所については、分割した形で質問の要約が与えられます。以前のタスクで「〔1〕」「〔2〕」などで表されていたそれぞれの質問ごとに、対応した要約を生成します。
  • 出力文字数の上限ではなく、実際の文字数が与えられます。 → 正解の文字数は与えない形に変更しました。

データセット

2001年以降の東京都議会（定例会）を対象とします。

  • 入力
    • 都議会だよりに含まれる、議員の質問の要約（JSON フォーマット）
    • 本会議の会議録に含まれる、議員・答弁者（説明員）の発言（JSON フォーマット）
  • 出力
    • 各質問に対応する答弁（説明）の簡潔な要約（JSON フォーマット）

評価

  • 自動評価
    • ROUGE-1-F_measure（Leader boardのスコアに用いる）
      • PoliInfo2 までは ROUGE-1-Recall
      • 模範解答は、議会だよりの Answer Summary
    • その他の ROUGE
  • 人手評価: 参加者間で相互評価（dry run で人手評価を試して修正）
    • Answer-type validity: 質問されたことに回答しているか（「〇〇できないか？」→「できる」or「できない」）
    • Content: 模範解答と比較して内容が一致しているか
    • Well-formed: 日本語として不自然な表現がないか
    • Total: 総合的に見た回答の適切さ
